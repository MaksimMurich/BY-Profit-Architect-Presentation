# Dashboard Demo — Технічне завдання
## Owner Dashboard: Flatbed Trucking Company

---

## 1. Мета документу

Описати структуру, логіку та UX/UI вимоги до демонстраційного дашборду власника тракової компанії (flatbed, ~35 траків). Документ є основою для побудови `dashboard-demo.html` — інтерактивного прототипу, що демонструє можливості системи потенційному клієнту.

---

## 2. Аудиторія

**Власник компанії:**
- Немає часу на довгі таблиці
- Приймає рішення на базі ключових цифр
- Хоче бачити відхилення, а не просто дані
- Переглядає переважно з десктопу
- Пріоритет: "де я зараз і де проблема"

**Контекст компанії:**
- 35 траків (flatbed / step deck)
- 3 диспетчери, ~30 активних водіїв
- Ключові витрати: паливо, зарплата, амортизація
- Основний показник ефективності: чистий прибуток на трак на тиждень

---

## 3. UX/UI Принципи

### 3.1 Ієрархія уваги
- **Рівень 1 (зверху):** KPI-картки — критичні показники станом на сьогодні
- **Рівень 2 (середина):** Тренди та рейтинги — що відбувалось і хто де
- **Рівень 3 (деталі):** Аналітичні графіки — для глибшого розуміння
- **Рівень 4 (бічна панель):** Алерти — що потребує дії прямо зараз

### 3.2 Кольорова семантика (строго витримується)
| Колір | Значення | Приклад |
|---|---|---|
| Зелений `#10b981` | Добре, зростання, топ | Прибуток вище норми |
| Синій `#3b82f6` | Нейтральний, інформативний | Завантаженість |
| Жовтий/Золотий `#f5b731` | Акцент, увага | Ключові метрики |
| Помаранчевий `#f59e0b` | Попередження | Середня ефективність |
| Червоний `#ef4444` | Проблема, спад | Збитковий водій |

### 3.3 Типи графіків — що використовуємо і чому
| Тип | Коли | Чому |
|---|---|---|
| Area chart (площинний) | Тренди в часі | Візуально показує обсяг і напрям |
| Horizontal bar | Рейтинги | Легко порівняти між собою, мітки читаються |
| Vertical bar | Порівняння по категоріях | Стандарт для лейнів, тижнів |
| Grouped bar | Порівняння кількох метрик | Диспетчери: виручка vs завантаження |
| Line chart | Порівняння трендів | CPM свій vs ринковий |
| Stacked bar | Склад цілого в часі | Типи вантажів по тижнях |

**Заборонені типи:** кругові (pie), кільцеві (donut), gauge (спідометри) — не читаються, не порівнюються, не дають динаміки.

### 3.4 Інтерактивність (обов'язково)
- **Hover на графік** → tooltip з точними значеннями
- **Анімація при завантаженні** → bars/lines "виростають" з нуля (300–600ms)
- **Hover на KPI картку** → легкий підйом (transform: translateY)
- **Клік на водія** → розгортається детальна картка (рейси, лейни, паливо)
- **Фільтр часу** → кнопки "Тиждень / Місяць / Квартал" перемикають дані

### 3.5 Дизайн-правила
- Не більше 6–8 елементів у зоні активної уваги одночасно
- Відступи між блоками — мінімум 16px
- Кожен графік має заголовок + одну підказку (що він показує)
- Числа на картках: великий шрифт (28–36px), жирний
- Дельта (зміна vs попередній період): маленька, зі стрілкою і кольором

## 13. Технічний Стек

| Компонент | Рішення |
|---|---|
| Графіки | ApexCharts (CDN) |
| Layout | CSS Grid + Flexbox |
| Анімації | ApexCharts built-in + CSS transitions |
| Шрифт | system-ui (без зовнішніх залежностей) |
| Іконки | Unicode символи або SVG inline |
| Дані | Hardcoded JS масиви (demo) |
